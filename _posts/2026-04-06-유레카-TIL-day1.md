## 2. Java 동작 원리 및 특징
* **플랫폼 독립성**: JVM(Java Virtual Machine)을 통해 OS 관계없이 동일 바이트코드 실행
* **컴파일 과정**: `.java` (소스코드) → `javac` (컴파일러) → `.class` (바이트코드) → `JVM` (실행)
* **G1GC (Garbage First GC)**: 메모리 관리를 위해 효율적으로 쓰레기를 수집하는 기본 방식
* **모니터링**: `jconsole` 활용하여 서버의 쓰레드 및 메모리(Heap/Non-Heap) 상태 시각적 확인 가능

## 3. 데이터 타입 및 연산자

### 데이터 타입
* **정수형**: 기본 타입 `int`. `byte`, `short` 변수는 연산 시 `int`로 자동 변환됨
* **실수형**: 기본 타입 `double`. `float` 선언 시 리터럴 뒤에 `f` 또는 `F` 접미사 필수
* **논리형**: `boolean` (true/false만 저장 가능, 0/1 사용 불가)

### 비트 연산자

| 연산자 | 기능 | 비고 |
| :--- | :--- | :--- |
| `>>` | 오른쪽 시프트 | 부호 비트 유지 (2의 n승으로 나눈 효과) |
| `>>>` | 오른쪽 시프트 | 왼쪽 빈자리를 0으로 채움 |
| `<<` | 왼쪽 시프트 | 오른쪽 빈자리를 0으로 채움 (2의 n승으로 곱한 효과) |

### 논리 연산자 (Short-circuit)
* `&&`, `||`: 앞의 항에서 결과 확정 시 뒤의 항은 실행하지 않음 (효율적 연산)

## 4. 제어문 및 메서드
* **Switch**: `score/10` 등을 활용한 조건 분기 및 `break` 사용
* **향상된 for문 (for-each)**: 배열/컬렉션 요소를 처음부터 끝까지 순차 추출하여 변수에 저장
* **Method**: 코드 재사용 최소 단위. 접근 제한자 및 리턴 타입 정의

## 오늘 실습 문제 : 정올 539번
**문제:** 100 이상의 정수가 입력될 때까지 합계와 평균(소수 첫째자리) 출력

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int k = 0;
        int sum = 0;
        
        while (true) {
            int n = sc.nextInt();
            k += 1;
            sum += n;
            if (n >= 100) break;
        }
        
        System.out.println(sum);
        System.out.printf("%.1f", (double)sum / k);
    }
}
```
