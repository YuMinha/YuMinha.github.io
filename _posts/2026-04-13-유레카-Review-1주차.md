## Chapter01. 자바 기초
### 바이트코드 파일, 자바 가상 머신(JVM)
- .java 파일을 작성한 후 컴파일을 해야한다.
- javac 명령어가 소스 파일을 컴파일하며 결과로 `.class`인 `바이트코드` 파일이 생성된다.

### JVM (Java Virtual Machine)
- 바이트코드 파일을 os가 이해하는 기계어로 번역하고 시행시킨다.
- JVM을 통해 os 관계없이 동일한 바이트 코드를 실행한다. => `플랫폼 독립성`
- JDK는 다르게 설치한다.

### G1GC (Garbage First GC)
- 메모리 관리를 위해 쓰레기를 효율적으로 수집하는 기본 방식

## Chapter02. 변수, 타입
### 데이터 타입
1. 정수
    - 기본 타입 `int`, `byte`, `short`
    - byte, short 타입은 변수로 연산시 int로 자동 변환됨.  
2. 실수
    - 기본 타입 `float`, `double`
    - float를 선언할 경우 f, F를 붙여야 한다. 
3. 논리
    - 기본 타입 `boolean`
    - true/ false만 저장가능하다. 0/ 1 불가능

### 자동 형변환
- 값의 범위가 작은 -> 큰 타입으로 대입될 때 발생
- byte -> short, char -> int -> long -> float -> double

### 타입 캐스팅
- 명시적으로, 강제적으로 형변환
- int -> byte
    ```java
    byte b1 = 3, b2 = 5;
    byte b3 = (byte)(b1+b2);
    ```
    : int 보다 작은 정수타입은 연산 시 자동으로 int로 변환됨. (byte)하지않으면 에러 발생

## Chapter05. 참조
### Java의 메모리 구조
- 메소드 영역 : 클래스 정보, 메소드 코드등 바이트코드파일의 내용이 저장되는 영역
- 힙 영역 : 객체
- 스택 영역 : 스레드 별로 운영, 기본 타입 변구, 참조 타입 변수가 생성 및 제거되는 영역

### 객체 생성 및 참조
1. **선언** : Stack에 변수 공간을 확보한다. 초기값은 null
2. **생성** : Heqp 영역에 객체 메모리 할당 및 생성 후 주소값 반환한다.
3. **대입** : 생성자 호출 후 반환된 주소값을 Stack의 변수에 저장한다.

- 비교 연산자 (==, !=)
  : 메모리의 값을 비교한다. 참조 타입 변수일 경우 참조값을 비교한다.
  (내용 비교하려면 equals, 기본 오브젝트는 ==, equals 동일)

### Pass By Value
: 전달할 때 변수에 저장된 `값`을 전달
- **Primitive** : 변수에 실제 값이 저장 = Pass By Value
- **Reference** : 변수에 참조 값이 저장 = Pass By Reference

### 배열 (Array)
- 같은 타입의 값을 관리
- 배열의 길이를 늘리거나 줄일 수 없다.
- Heap 영역에 생성되고 new 연산자로 처음 생성하면 기본값으로 초기화 된다.
  - 정수 : 0
  - 실수 : 0.0
  - 논리 : false
  - 참조 : null
- 다차원 배열 : 일차원 배열 이어 놓은 것
  - 다른 크기의 배열로 배열을 만드려면 
    ```java
      int[][] arr = new int[3][];
      arr[0] = new int[5];
      arr[1] = new int[2];
      arr[2] = new int[10];
    ```
- 배열 복사
  : 새로운 배열을 생성해 내용을 복사해야 한다.

## Chapter06. 객체지향
### 객체 (Object)
- 현실 세계의 모든 것 (객체 지향에서는 현실 세계의 모든 것들이 프로그램의 대상)
- 속성과 행위(메서드)를 가지고 있음
- **재사용의 단위**

### Key Abstraction
- 무수히 많은 객체 중에서 프로그램에 필요한 객체를 선정하고, 객체의 속성과 기능 중에서 필요한 것만 추출하여 클래스를 정의하는 과정
- 명세서를 통해 명사(데이터: 속성, 객체)와 동사(기능, 관계)를 추출함

### 객체지향 프로그래밍 특징
- **캡슐화 (Encapsulation)**: 속성과 메서드를 하나로 묶고 접근을 제한하여 데이터 보호 및 은닉(`decouple`, 유지보수성 향상)
- **상속 (Inheritance)**: 재사용성과 수정성을 높이기 위한 특징
- **다형성 (Polymorphism)**: 재사용성과 수정성을 높이기 위한 특징

### 클래스 (Class)
- 객체의 속성과 행위를 선언하여 객체를 생성할 수 있는 대상
- 클래스 코드가 객체 생성을 통해 메모리에 올라가므로 매번 선언할 필요 없이 재사용 가능

### 생성자 (Constructor)
- **기본 생성자 (Default Constructor)**: 클래스 내부에 생성자가 없으면 컴파일러가 인자가 없는 생성자를 자동으로 선언함
- **생성자 오버로딩 (Overloading)**: 이름은 같으나 인자의 개수, 타입, 순서를 다르게 하여 여러 개 선언하는 것

### this 키워드
- 클래스로부터 생성된 객체 중 **현재 사용 중인 객체**를 참조
1. 속성과 로컬변수 이름이 같을 때 구별하기 위해 사용 (`this.속성명`)
2. 코드 재사용을 위해 인자가 다른 생성자를 호출할 때 사용 (`this(...)`) - 반드시 생성자 첫 번째 명령에서만 호출 가능

### 메소드 오버로딩 (Method Overloading)
- 동일한 기능의 함수가 인자가 달라서 여러 개 선언해야 할 때 이름을 동일하게 선언함
- 인자의 개수, 타입, 순서로 메서드를 구별하여 호출 시 편의성을 제공하고 유지보수성을 높임

### 접근 제한자 (Access Modifier)
- 클래스, 메서드, 속성에 대한 접근을 제한함
| 제한자 | 접근 범위 |
| :--- | :--- |
| **public** | 접근 제한이 전혀 없음 |
| **protected** | 상속 관계는 제한 없음, 상속이 아니면 같은 패키지만 가능 |
| **default(생략)** | 같은 패키지 내에서만 접근 가능 |
| **private** | 선언한 클래스 내부에서만 접근 가능 |

### Getter / Setter
- 캡슐화된 데이터에 접근하기 위해 제공되는 public 메서드
- 데이터 보호 및 유효성 검사(예: 잘못된 날짜나 급여 설정 방지)를 수행함


## Chapter 07. 객체지향 심화 (상속과 다형성)

### static과 final
* **static**:
    * 클래스가 로딩될 때 메모리(static 영역)에 한 번 할당되며, 객체 생성 없이 클래스명으로 접근 가능
    * **static 메서드** 내에서는 인스턴스(non-static) 속성이나 메서드에 접근할 수 없음
* **final**:
    * **클래스**: 상속 불가 (예: `String`).
    * **메서드**: 오버라이드 금지
    * **변수**: 값 변경 불가 (상수).
    * **Blank Final**: 선언 시 초기화하지 않고 생성자에서 객체마다 다른 고정값을 할당하는 방식
  
### 상속 (Inheritance)
* **개념**: 부모 클래스의 속성과 메서드를 자식 클래스가 물려받는 것
* **super 키워드**:
    * `super()`: 부모의 생성자를 호출하며, 반드시 자식 생성자의 **첫 번째 명령**이어야 함
    * `super.메서드명()`: 재정의(Override)에 의해 가려진 부모의 메서드나 속성에 접근할 때 사용

### 메서드 오버라이드 (Method Override)
* **정의**: 상속받은 메서드와 동일한 이름, 인자, 리턴 타입을 가진 메서드를 자식 클래스에서 새롭게 정의하는 것
* **규칙**:
    * 이름과 인자 리스트는 반드시 같아야 함
    * 리턴 타입: 기본 타입은 같아야 하며, 참조 타입은 부모 타입이거나 그 자식(Sub) 타입일 수 있음
    * 접근 제한자: 부모보다 **같거나 더 넓은 범위**로만 가능
    * 예외(Exception): 부모보다 더 구체적인(Sub) 예외를 던지거나 아무 예외도 던지지 않을 수 있음

### 다형성 (Polymorphism) ★핵심
* **개념**: 부모 타입의 변수로 자식 타입의 객체를 참조할 수 있는 성질
* **형변환 (Casting)**:
    * **Up-casting**: 자식 → 부모 타입 변환. 자동으로 이루어짐
    * **Down-casting**: 부모 → 자식 타입 변환. 반드시 명시적으로 작성해야 하며, 실제 참조 중인 객체 타입이 아닐 경우 `ClassCastException` 발생
* **instanceof**: 객체가 특정 타입인지 검사하는 연산자. 형검사 시 반드시 **자식(Sub) 타입부터** 순서대로 검사해야 함
* **Virtual Invocation (Dynamic Binding)**: 다형성 관계에서 메서드 호출 시, 변수의 타입이 아닌 **실제 생성된 객체의 Override된 메서드**가 호출되는 현상 
    * 단, **속성(Field)**과 **static 메서드**는 참조 변수의 타입을 따름

### 추상 클래스 (Abstract Class)
* **특징**:
    * `abstract` 키워드를 사용하며, **객체 생성이 불가능**함
    * 추상 메서드가 하나라도 있으면 반드시 추상 클래스로 선언해야 함
    * 자식 클래스에서 추상 메서드를 반드시 Override 하도록 강제함
* **목적**: 상속 전용으로 사용하며, 자식들의 공통 규격을 설계함

### Object 클래스의 주요 메서드
* **equals(Object obj)**: 객체의 내용을 비교. 기본적으로 `==`과 같으므로 내용 비교를 위해 반드시 Override 필요 10].
* **hashCode()**: 객체의 고유 정수값 리턴. Set 계열 자료구조 사용 시 Override 권장
* **toString()**: 객체 정보를 문자열로 리턴. 객체를 출력하거나 문자열 결합 시 자동 호출됨
* **clone()**: 객체 복제. `Cloneable` 인터페이스를 구현해야 정상 동작함
* **finalize()**: GC가 메모리에서 객체를 제거할 때 호출되나 성능 저하를 일으킬 수 있음


## Chapter 08. 인터페이스와 디자인 패턴

### 1. 인터페이스 (Interface)
* **구성 요소**: 오직 **상수(Constant)**와 **추상 메서드(Abstract Method)**로만 구성된다. 
    * **상수**: 별도 선언이 없어도 `public static final`이 컴파일 시 자동 추가  
    * **추상 메서드**: 별도 선언이 없어도 `public abstract`가 컴파일 시 자동 추가
* **특징**:
    * **다중 상속 가능**: 클래스와 달리 여러 인터페이스를 한꺼번에 구현(`implements`)할 수 있어 다형성 확장에 유리
    * **응집도(Cohesion)**: 관련 기능들을 밀접하게 모아 구현함으로써 프로그램의 책임 소재를 명확히 함
    * **JDK 8 이후 기능**: `default` 메서드(구현부가 있는 메서드)와 `static` 메서드를 포함할 수 있음 

### 2. 열거형 (Enum)
* **상수 관리의 이점**: 기존 인터페이스 상수를 사용할 때 발생하는 **값 중복 비교 오류**를 방지
    * 서로 다른 Enum 타입끼리는 비교가 불가능하여 타입 안정성이 보장 
* **기능**:
    * `name()`: 열거 상수의 이름을 문자열로 리턴
    * `ordinal()`: 열거 상수의 순번(0부터 시작)을 리턴
    * **생성자와 필드**: Enum 내부에 필드와 생성자를 선언하여 "봄", "여름" 등 추가적인 데이터를 매핑할 수 있다

### 3. 주요 디자인 패턴
#### ① 싱글톤 패턴 (Singleton Pattern)
* **목적**: 클래스로부터 생성되는 **객체가 오직 하나**임을 보장. 
* **규칙**
    1. 클래스 내부에 자신의 타입을 `static` 속성으로 선언
    2. 생성자를 `private`으로 선언하여 외부 생성을 차단
    3. 객체를 전달하는 `public static` 메서드(주로 `getInstance`)를 제공
* **LazyHolder 기법**: `getInstance()`가 처음 호출될 때 객체를 생성하는 방식으로, 내부 `static nested class`를 사용하여 **Thread-safe**하게 구현

#### ② 팩토리 패턴 (Factory Pattern)
* **개념**: 객체 생성 로직을 별도의 클래스(`Factory`)로 분리하여 관리
* **효과**: 메인 코드(`FactoryTest`)를 수정하지 않고도 `Factory`의 반환 객체만 바꾸면(예: `Phone` → `SmartPhone`) 프로그램의 동작을 변경할 수 있어 결합도가 낮아짐

### 4. 코드 동작 흐름 (Factory 실습 중심)
* **Product (추상 클래스)**: `service(Command)`를 추상 메서드로 가져 모든 제품의 공통 규격을 정의
* **Command (인터페이스)**: 제품에 내릴 명령들의 규격. `PhoneCommand` 등이 이를 구현
* **Polymorphism**: `Product p = ProductFactory.getInstance();`를 통해 실제 객체가 무엇인지 몰라도 `p.service()`를 호출할 수 있다. (다형성 및 동적 바인딩 적용)

## **Chapter12. 베이스 모듈**

### **Wrapper 클래스**
* **개념**: 기본 타입(primitive)의 데이터를 객체로 다루기 위해 포장하는 클래스
* **Box & Unbox**: 
    * **Auto Boxing**: 기본 타입 값을 래퍼 객체에 대입 시 자동 객체화 (`Integer i = 10;`)
    * **Auto Unboxing**: 래퍼 객체를 기본 타입 변수에 대입 시 자동 값 추출 (`int n = i;`)
* **문자열 변환**: `Integer.parseInt()`, `Double.parseDouble()` 등을 사용하여 문자열을 해당 타입으로 변환한다
    * 잘못된 포맷의 문자열을 변환하려 하면 `NumberFormatException`이 발생하므로 주의해야 한다

### **String 클래스**
* **객체 생성 방식**:
    * **Literal 방식**: `"..."`로 생성하며 **Literal Pool**에 저장되어 동일 문자열은 재사용된다
    * **new 연산자**: **Heap** 영역에 매번 새로운 객체를 생성한다
* **특징**:
    * **불변성**: 문자열 연산(`+`, `replace` 등) 결과는 기존 객체를 바꾸는 것이 아니라 새로운 객체를 생성하는 것이다
    * **비교**: 주소 비교(`==`)가 아닌 **내용 비교(`equals()`)**를 사용해야 안전하다



### **StringBuilder & StringTokenizer**
* **StringBuilder**: 문자열을 빈번하게 수정할 때 사용하며, 새로운 객체를 만들지 않고 내부 버퍼를 이용해 성능을 높인다.
* **StringTokenizer**: 구분자(Delimiter)를 기준으로 문자열을 분리할 때 사용한다
    * `split()`은 정규표현식을 사용할 수 있고, `StringTokenizer`는 단순 문자로 빠르게 분리할 때 유리하다

### **Math & Random**
* **Math**: 수학 계산을 위한 static 메서드를 제공한다
    * `ceil()`: 올림, `floor()`: 내림, `round()`: 반올림
    * `random()`: 0.0 이상 1.0 미만의 실수를 리턴한다
* **Random**: 시드(Seed) 값을 설정할 수 있으며, 시드가 같으면 항상 동일한 난수를 발생시킨다

### **Date & Calendar**
* **Date**: 현재 날짜와 시간 정보를 저장하며, `SimpleDateFormat`을 통해 원하는 포맷으로 출력 가능하다
* **Calendar**: 추상 클래스로 `getInstance()`를 통해 객체를 얻는다
    * **주의**: `Calendar.MONTH`는 **0부터 시작**하므로 실제 월을 구하려면 반드시 **+1**을 해야 한다
    * 요일 정보는 `DAY_OF_WEEK` 상수로 관리하며 일요일(1)부터 시작한다

### **Pattern (정규표현식)**
* 특정한 규칙을 가진 문자열의 집합을 표현하는 형식이다
* `Pattern.matches(regExp, data)`를 통해 전화번호나 이메일 양식의 유효성을 검사한다

### **Record (Java 14+)**
* **목적**: 데이터 전달을 위한 **DTO**를 간결하게 작성하기 위해 사용한다
* **특징**: 필드 선언만으로 **getter, toString(), equals(), hashCode()**가 자동으로 생성된다
    * 데이터를 읽을 때 `r1.name()`과 같이 필드명과 동일한 메서드를 사용한다


## **Chapter13. 제네릭 (Generic)**

### **제네릭 (Generic) 개념**
* **정의**: 클래스, 인터페이스, 메서드를 정의할 때 타입을 파라미터로 사용하여 다양한 타입에 대응할 수 있도록 하는 기능이다.
* **특징**:
    * 컴파일 시 강한 타입 체크를 통해 실행 시 발생할 수 있는 오류(`ClassCastException`)를 방지한다.
    * 불필요한 타입 변환(Casting)을 제거하여 프로그램의 성능과 가독성을 높인다.

### **제네릭 타입과 메소드**
* **제네릭 클래스**: `class Product<T, M> {}`와 같이 선언하며, 객체 생성 시 구체적인 타입을 지정한다.
* **제네릭 메소드**: 리턴 타입 앞에 `<T>`를 붙여 선언하며, 호출 시 매개값의 타입에 따라 타입 파라미터가 결정된다.
    * 형식: `public <T> Box<T> boxing(T t) { ... }`

### **제한된 타입 파라미터 (Bounded Generic)**
* **개념**: `extends` 키워드를 사용하여 타입 파라미터에 지정할 수 있는 타입의 상한을 정하는 것이다.
* **특징**: 지정한 클래스 또는 그 하위 클래스만 타입으로 사용할 수 있다.
    * 예: `<T extends Number>`는 `Integer`, `Double` 등 숫자 관련 타입만 가능하다.

### **와일드카드 (Wildcard) 타입 파라미터**
* **`<?>` (Unbounded Wildcards)**: 모든 타입을 사용할 수 있는 제한 없는 와일드카드이다.
* **`<? extends T>` (Upper Bounded Wildcards)**: **T** 타입 또는 그 **하위(자식)** 타입만 가능하도록 제한한다.
    * 예: `<? extends Student>` → Student, HighStudent 등 가능
* **`<? super T>` (Lower Bounded Wildcards)**: **T** 타입 또는 그 **상위(부모)** 타입만 가능하도록 제한한다.
    * 예: `<? super Worker>` → Worker, Person 등 가능


## **Chapter15. 컬렉션 프레임워크 (Collection Framework)**

### **1. List 컬렉션**
* **특징**:
    * 데이터를 저장한 순서대로 유지한다.
    * 인덱스(Index)로 관리하며, 중복 저장이 가능하다.
    * `indexOf()`, `contains()`, `remove()` 등의 메서드 실행 시 `equals()`를 호출하여 객체의 내용을 비교한다.
* **주요 클래스**: `ArrayList`, `LinkedList`.
* **주의**: 인덱스 범위를 벗어난 위치에 접근 시 `IndexOutOfBoundsException`이 발생한다.

### **2. Set 컬렉션**
* **특징**:
    * 동일한 객체를 중복해서 저장할 수 없어 데이터의 유니크성을 보장한다.
    * 저장 순서가 유지되지 않으며, 인덱스가 존재하지 않는다.
    * 객체 중복 여부를 판단하기 위해 `hashCode()`와 `equals()`를 모두 호출하여 비교한다.
* **데이터 추출**: 인덱스가 없으므로 `Iterator` 인터페이스를 사용하거나 향상된 for문을 사용하여 데이터를 순회한다.


### **3. Map 컬렉션**
* **특징**:
    * 키(Key)와 값(Value)의 쌍(Pair)으로 데이터를 관리하며, 키는 중복될 수 없다.
    * 검색 속도가 매우 빠르며, 동일한 키로 새로운 값을 저장하면 기존 값이 덮어쓰여진다.
* **주요 메서드**:
    * `keySet()`: 저장된 모든 키를 `Set` 객체에 담아 리턴하여 키 반복 처리가 가능하다.
    * `values()`: 저장된 모든 값을 `Collection` 타입으로 리턴한다.
    * `get(key)` / `remove(key)`: 키를 통해 값을 가져오거나 삭제한다.

### **4. 컬렉션과 제네릭 (Generic)**
* **적용**: 컬렉션 선언 및 생성 시 저장할 객체의 타입을 지정하여 사용한다.
* **이점**:
    * 컴파일 단계에서 타입 체크가 이루어져 지정한 타입 외의 저장을 방지한다.
    * 데이터를 꺼낼 때 별도의 형변환(Casting) 과정 없이 바로 해당 타입으로 사용할 수 있다.
    * 제네릭을 부모 타입으로 설정하면 다형성에 의해 하위(Sub) 타입 객체들도 저장이 가능하다.

## **Chapter16. 람다식과 함수형 인터페이스**

### **1. 함수형 인터페이스 (Functional Interface)**
* **정의**: 단 하나의 추상 메서드만을 가지는 인터페이스를 의미한다.
* **구현 방식**: 
    * **클래스 구현**: 인터페이스를 상속받은 별도의 클래스를 정의하여 사용한다.
    * **익명 내부 클래스**: 선언과 동시에 객체를 생성하여 일회성으로 사용한다.
    * **람다식 (Lambda)**: 함수형 인터페이스의 추상 메서드를 간결한 코드로 직접 구현한다.

### **2. 람다식 (Lambda Expression)**
* **특징**: 객체 지향 언어인 자바에서 함수형 프로그래밍 스타일을 지원하기 위해 도입되었다.
* **사용 조건**: 반드시 메서드가 하나인 **함수형 인터페이스**를 대상으로만 작성이 가능하다.
* **문법**: `(매개변수) -> { 실행문; }` 형태로 작성하며, 코드가 간결해지고 가독성이 향상된다.

### **3. 객체 정렬과 Comparator**
* **Arrays.sort()**: 기본 타입 배열은 오름차순으로 자동 정렬되나, 객체 배열은 정렬 기준이 필요하다.
* **Comparator 인터페이스**: 정렬 기준을 사용자 정의할 때 사용하며, 람다식을 활용해 간결하게 구현할 수 있다.
    * **오름차순**: `(e1, e2) -> e1.getSalary() - e2.getSalary()` (앞의 값이 크면 양수 반환).
    * **내림차순**: `(e1, e2) -> e2.getSalary() - e1.getSalary()` (뒤의 값이 크면 양수 반환).
* **String 정렬**: 문자열의 경우 `compareTo()` 메서드를 활용하여 오름차순/내림차순 정렬을 수행한다.
