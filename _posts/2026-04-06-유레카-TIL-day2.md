## 1. 배열 (Array)
### 1-1. 배열의 특징 및 생성
* 동일한 데이터 타입의 집합. 크기 변경 불가능.
* Heap 영역에 생성되며 생성 시 타입별 기본값으로 자동 초기화됨. (int: 0, 참조형: null 등)
* `Arrays.toString()`을 통해 배열 내용을 쉽게 출력 가능.

```java
// 코드 예시 (ArrayTest01.java)
int[] a1;
a1 = new int[3]; // Heap에 생성, {0, 0, 0} 초기화

int[] a3 = {0, 1, 2, 3, 4}; // 선언, 생성, 할당 동시 진행
````

### 1-2. 다차원 배열

  * 자바는 가변 배열(행마다 열의 크기가 다른 배열) 지원.
  * ( 자바는 배열을 힙 영역에서 생성하고 쓴다. 포인터가 없어서 다차원 배열도 배열 여러 개를 사용하는 것이다. )

<!-- end list -->

```java
// 코드 예시 (ArrayTest03.java)
int[][] a2 = new int[3][]; // 행 크기만 지정
a2[0] = new int[5]; // 각 행별로 열 크기 다르게 생성 가능
a2[1] = new int[1];
```

### 1-3. 배열 복사

  * 크기 변경이 불가하므로, 큰 배열이 필요하면 새로 생성 후 복사해야 함.

<!-- end list -->

1.  `System.arraycopy()`: 원본과 목적지, 위치, 개수를 지정하는 네이티브 메서드. 효율이 좋음.
2.  `Arrays.copyOf()`, `Arrays.copyOfRange()`: 새로운 배열을 생성하여 복사본 리턴.

## 2\. 객체 지향 프로그래밍

### 2-1. 현실 세계와 객체 (Object)

  * **정의**: 현실 세계에 존재하는 모든 것. 구분 가능한 대상.
  * **구성**: 데이터(속성/필드) + 행위(메서드).
  * **Key Abstraction (핵심 추상화)**: 요구사항 정의서(Use Case 명세서) 등의 문서에서 명사(Data → 속성)와 동사(행위 → 메서드, 관계)를 추출하여 객체를 설계하는 과정.
  * ![alt text](\assets\images\2026-04-06-유레카-TIL-day2\image1.png)

### 2-2. 프로그래밍 비교

  * **절차적 프로그래밍**: 코드의 재사용성이 낮음.
  * **구조적 프로그래밍**: 함수(메서드) 단위의 모듈화로 재사용성은 높였으나, 유지보수가 어려움.
  * **객체 지향 프로그래밍**: 모듈화 및 재사용성이 높고, 상속/캡슐화/**다형성**을 통해 유지보수성이 크게 향상됨.

### 2-3. Java와 Memory 구조 (런타임 데이터 영역)
  ![alt text](\assets\images\2026-04-06-유레카-TIL-day2\image2.png)

  * **Method Area**: `MyType.class`와 같은 클래스 정보, 메서드 코드 저장.
  * **Stack**: 스레드별로 운영됨. 기본 타입 변수 및 참조 타입 변수(`m`, `cust` 등)의 참조값(주소) 저장.
  * **Heap**: `new` 연산자로 생성된 객체 및 배열이 저장되는 영역.
      * 영역 구분: Eden, S1, Old 영역으로 나뉘며 G1GC 등의 가비지 컬렉터가 관리함.
      * 포인터가 없는 자바 구조상 다차원 배열 구현 등을 위해 배열을 힙에 생성함.

## 3\. Java의 값 전달 방식: Pass By Value

Java는 메서드 호출 시 인자 전달 방식으로 오직 **Pass By Value**만 사용함. 변수에 저장된 **값**을 복사해서 전달함.

### Primitive vs Reference 타입 전달의 차이
  ![alt text](\assets\images\2026-04-06-유레카-TIL-day2\image3.png)

  * **Primitive (기본형)**: 변수에 실제 값이 저장되어 있으므로, **실제 값**이 복사되어 전달됨. 호출된 메서드에서 매개변수 값을 바꿔도 원본 변수는 변하지 않음.
  * **Reference (참조형)**: 변수에 객체의 \*\*참조값(주소)\*\*이 저장되어 있으므로, **주소값**이 복사되어 전달됨.
      * **결과적으로 주소를 공유**하게 되므로, 호출된 메서드에서 매개변수(배열 등)의 **내용을 변경하면 원본 객체도 변경됨**.

## 4\. 참조 타입과 메커니즘

### 4-1. 객체 생성 및 참조
![alt text](\assets\images\2026-04-06-유레카-TIL-day2\image4.png)

1.  **선언**: `Customer cust = null;` (Stack에 변수 공간 확보, 초기값 null).
2.  **생성**: `new Customer(...);` (Heap에 객체 메모리 할당 및 기본값 초기화, 주소값 반환).
3.  **호출/대입**: 생성자 호출 후 반환된 주소값(`0x33`)을 Stack의 `cust` 변수에 저장.
4.  **this**: 생성된 객체 자신을 참조하는 키워드. 속성과 로컬 변수의 이름이 같을 때 구분(`this.name = name`)하거나 다른 생성자를 호출(`this(...)`)할 때 사용.

### 4-2. 비교 연산자 (`==`, `!=`) vs `equals()` 메서드
![alt text](\assets\images\2026-04-06-유레카-TIL-day2\image5.png)

  * **`==`, `!=`**: **메모리의 값**을 비교.
      * 참조 타입 변수에서는 \*\*객체의 번지(참조값)\*\*를 비교함.
      * `MyType m1 = new MyType(); MyType m2 = new MyType();` 일 때, `m1 == m2`는 `false` (서로 다른 주소).
  * **`equals()`**: 객체의 **내용**을 비교하기 위해 재정의하여 사용. (기본 Object의 equals는 `==`와 동일).

## 오늘 실습 문제 : 정올 561번

10개의 정수를 입력받아 100 미만의 수 중 가장 큰 수와 100 이상의 수 중 가장 작은 수를 출력하는 프로그램을 작성하시오.

(입력되는 정수의 범위는 1이상 10,000 미만이다. 만약 해당하는 수가 없을 때에는 100 을 출력한다.)

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int[] nums = new int[10];

        for(int i = 0; i<10; i++){
            nums[i]=sc.nextInt();
        }

        int a=0;
        int b=10000;
        for(int i = 0; i<10; i++){
            if(nums[i]<100 && nums[i]>a) a=nums[i];
            else if(nums[i]>=100 && nums[i]<b) b=nums[i];
        }
        System.out.println((a>0?a:100)+" "+(b<10000?b:100));
    }
}
```
